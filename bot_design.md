# تصميم بوت تأجير النطاقات الفرعية

## 1. هيكل قاعدة البيانات (SQLite)
- **users**: (id, telegram_id, username, balance, is_admin)
- **rentals**: (id, user_id, subdomain, html_content, start_date, end_date, status)
- **settings**: (key, value) - لتخزين الأسعار والإعدادات العامة.

## 2. واجهة المستخدم (تليجرام)
- **/start**: رسالة الترحيب والقائمة الرئيسية.
- **تأجير دومين**: يطلب الاسم المطلوب وكود HTML.
- **حسابي**: عرض الرصيد والإيجارات النشطة.
- **شحن الرصيد**: (محاكاة أو تعليمات).
- **لوحة التحكم (للمسؤول)**: تغيير الأسعار، إدارة المستخدمين.

## 3. آلية العمل التقنية
- استخدام **Flask** مع **Blueprint** أو **Dynamic Routing** للتعامل مع النطاقات الفرعية.
- بما أننا على استضافة مجانية (مثل Render أو PythonAnywhere)، سنستخدم مساراً مثل `domain.com/u/subdomain` أو إذا كانت الاستضافة تدعم Wildcard سنستخدم `subdomain.domain.com`.
- نظراً لصعوبة الحصول على Wildcard DNS مجاني تماماً مع SSL بشكل تلقائي وفوري، سنعتمد في النسخة الأولى على نظام المسارات `host.com/s/subdomain` مع إمكانية التوسع.

## 4. خطة التنفيذ
1. برمجة البوت باستخدام `python-telegram-bot`.
2. برمجة خادم الويب باستخدام `Flask`.
3. دمج الاثنين في تطبيق واحد (أو منفصلين حسب الاستضافة).
4. الرفع على **Render** أو **PythonAnywhere**.
